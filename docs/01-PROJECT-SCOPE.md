# 01. Project Scope

## What CIRRUS Is

CIRRUS is a real-time playback optimization engine for damaged or underperforming playback chains.

Its purpose is not just to "make audio brighter" or "make everything wider." Its real aim is to improve how audio is perceived as a source in front of the listener:

1. reduce masking
2. reduce roughness and haze
3. stabilize center image
4. improve source focus
5. preserve listenability under real-world playback constraints

## Primary Use Cases

1. Browser media playback
   - streaming services
   - video platforms
   - compressed speech and music
2. Windows system-wide playback
   - desktop audio
   - media players
   - everyday speakers and headphones
3. Low-quality or compromised sources
   - lossy playback
   - harsh clipping
   - spatial collapse
   - dull or foggy presentation

## Non-Goals

CIRRUS is not primarily trying to be:

1. a conventional EQ
2. a generic widening effect
3. an obvious exciter
4. a fake reverb enhancer
5. a "make everything huge at all costs" consumer effect

## Product Philosophy

The project sits between restoration and presentation:

- it should recover useful structure when possible
- it may also improve presentation when strict restoration is impossible

That means CIRRUS must always manage a tension between:

1. truthful signal recovery
2. premium-system presentation

The project should never pretend those are always the same thing.

## Current Strategic Rule

If a technically cleaner design destroys the current core listening strengths, it is not an upgrade.

Core listening strengths are:

1. front-focused source placement
2. strong center image
3. reduced haze and roughness
4. premium-system presentation
5. low listening fatigue
