# 09. Licensing and Contributions

## Public License

This repository is licensed under:

`GNU Affero General Public License v3.0 or later`

That means:

1. the public code stays open under AGPL terms
2. modifications and qualifying network use must respect AGPL obligations
3. the repository remains open-source in its public form

See:

- [../LICENSE](../LICENSE)

## Commercial Licensing

Commercial licensing is available for users who do not want AGPL obligations.

Typical examples:

1. proprietary redistribution
2. OEM integration
3. closed-source internal products
4. paid services that do not want to release corresponding source

See:

- [../COMMERCIAL-LICENSE.md](../COMMERCIAL-LICENSE.md)

## Contribution Policy

The project welcomes contributions, but it is not community-owned.

It is an owner-led project with a commercial licensing path.

For repository governance and contribution relicensing purposes, the Project Owner is **Howard Chen**.

To keep that path viable, accepted contributors must agree that:

1. their contribution may remain in the AGPL repository
2. Howard Chen, as Project Owner, may also use and relicense that contribution commercially
3. contributors are credited, but repository stewardship stays with Howard Chen, as Project Owner

See:

- [../CONTRIBUTING.md](../CONTRIBUTING.md)
- [../CLA.md](../CLA.md)
- [../CONTRIBUTORS.md](../CONTRIBUTORS.md)

## Practical Summary

If you want to:

1. use CIRRUS as open-source software and comply with AGPL
   - use the public repository
2. use CIRRUS in a closed-source or proprietary setting
   - obtain a commercial license
3. contribute code back
   - read and accept the project contribution terms first
